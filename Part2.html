Project 2: Heatwave analysis for Italy¶

ABDILLAHI AHMED Mariam, EL HADAD Chaïma, HILLAH Kelly

Objective: We the examine consequences of the summer 2022 heatwave for Italy.

This project is loosely based on Ballester et al. (2023) who conduct a quantitative analysis of the mortality-temperature relationship using the NUTS3 spatial resolution (both for mortality figures and ERA5 temperature scans). Below, we use mortality data for the whole of Italy, so results will differ from those of Ballester et al. (2023).

  • Ballester , J. et al. (2023). Heat-related mortality in Europe during the summer of 2022. Nature Medicine.

1. Preliminaries: The temperature data for Italy¶

Task 1.1: Generate the ERA5 data extract for Italy using the API of daily-hourly data starting in 2015. As a consequence, the reference period for our analysis will be 2015-2020, when we assess the situation in the summer of 2022. Recall that the coordinates are (anti-clockwise): [N,W,S,E]. The bounding box for Italy is [46,9,37,18.5,]

We used the Copernicus Climate Data Store (CDS) API via the cdsapi library to download hourly ERA5 reanalysis temperature data for Italy. The variable downloaded is 2-meter surface temperature (2m_temperature) for the years 2015–2022, covering all months, days, and hours. The spatial bounding box [46, 9, 37, 18.5] in [N, W, S, E] format covers the Italian peninsula. The data was saved as a NetCDF file and the temperature was converted from Kelvin to Celsius by subtracting 273.15. Days 31 were removed at the loading stage to avoid invalid date issues downstream.

# Aggregate hourly -> daily means
year_month_day_mean = t2m_C.groupby(['time.year', 'time.month', 'time.day']).mean()

# Aggregate spatially with cosine latitude weighting
weights = np.cos(np.deg2rad(year_month_day_mean.latitude))
weights = weights.fillna(0)
weights.name = "weights"
year_month_day_mean_weighted = year_month_day_mean.weighted(weights)
Italy_t2m = year_month_day_mean_weighted.mean(["longitude", "latitude"])

The hourly data was aggregated to daily means using groupby on year, month, and day. A cosine latitude weighting was applied before taking the spatial mean, since grid cells closer to the equator cover more surface area than those near the poles. This gives a single daily mean temperature representative of all of Italy. The map above shows the spatial temperature distribution on 03.08.2022 — the peak of the heatwave — with most of the peninsula showing temperatures between 22–28°C. The daily series was then aggregated to weekly means and saved as a CSV file with columns year, week, and t2m.

2. The Mortality Data¶
Task 2.1: Use of the   World Mortality Dataset compiled by Ariel Karlinsky and Dmitry Kobak that records all-cause mortality at a weekly level. Filter Italy, which has iso3c code "ITA".


First, we do it in R langage.


Filtering for Italy (iso3c == "ITA") gives 505 weeks of data from 2015 onwards, with columns for year, week, mortality (deaths), and start_date. All-cause mortality is used rather than cause-specific mortality because heat-related deaths are often not directly attributed to heat on death certificates.

3. A simple time-series model of excess mortality¶
Task 3: Next, estimate a simple time-series model of excess mortality for the period before 2020-03-01, which only considers a linear year effect and week fixed effects. Then use the model to predict mortality for the summer of 2022, and compute the measure of excess mortality (i.e. observed - predicted weekly mortality from all causes).


To estimate excess mortality during the summer of 2022, we first build a baseline model using data before 2020-03-01, which coincide with the  pre-COVID period. The model is a simple linear regression of the form:


$$mortality = \beta_0 + \beta_1 \cdot year + \sum_{w=1}^{52} \gamma_w \cdot \mathbb{1}[week = w] + \varepsilon$$


It includes two components:


    

  • Linear year effect ($\beta_1 \cdot year$): captures the long-run downward trend in mortality over time
    • 

  • Week fixed effects ($\gamma_w$): captures seasonal patterns — mortality is systematically higher in winter and lower in summer
    • 



We use this model to predict what mortality would have been in the summer of 2022 under normal conditions.


Excess mortality is defined as:
$$\text{excess deaths} = \text{observed mortality} - \text{predicted mortality}$$


We implement this model twice:


    

  • First in R using lm() with as.factor(week) for the week fixed effects.
    •
